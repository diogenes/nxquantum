# Skill: Docs and RFC

## Goal

Keep architectural intent explicit and discoverable.

## Workflow

1. Update `docs/architecture.md` when boundaries or responsibilities change.
2. Add an ADR in `docs/adr/` for significant decisions.
3. Update `README.md` and `CONTRIBUTING.md` for developer-facing impact.
4. Ensure examples align with real public API.

## Done Criteria

- Decision rationale is documented.
- New contributors can discover setup and workflow in under 10 minutes.
- API examples are coherent with current code.
